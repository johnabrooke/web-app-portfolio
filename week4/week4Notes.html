Ch.8 Forms

Accessing Form Elements

  • document.forms: legacy DOM property that returns an HTML collection of all the forms in the document in the order they appear in the markup, a collection will be returned even if there is only one form; it will be a collection of one. (ex. documetn.forms[0]; returns the first form object)
  • Instead of using a numerical index, we can use the name attribute to identify a form: const form = document.forms.search; Be careful referencing elements in this way, however. If the form had the same name as any properties or methods of the document.forms object, such as 'submit', for example, that property or method would be referenced instead of the <form> element. This is unlikely to happen, as the document.form object doesn't have many properties or methods, but it is something to be aware of. To avoid this, square bracket notation can be used (this is also required if the formâ€™s name attribute contains any invalid characters, such as spaces or dashes): const form = document.forms['search'];

Ch.12 Object Oriented Programming

Constructor Property

  • All objects have a constructor property.
  • The constructor property returns the constructor function that created it.
  • When a new object is created using an object literal, the Objectconstructor (one of the built-in constructor functions in JavaScript) is being used in the background.
  • The constructor property can be used to instatiate a copy of an object. This is helpful when the constuctor name is unknown because there's no need to reference the constructor function or class declaration directly. For example:
    const greenDice = new redDice.constructor(10);
